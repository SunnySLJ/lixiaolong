## 🚀 测试步骤

### 步骤 1：登录抖音（手动）

打开命令行，运行：

```bash
cd C:\Users\爽爽\.openclaw\workspace\douyin-auto-publish
python scripts/quick_login.py
```

**会发生什么：**
1. 自动打开 Chrome 浏览器
2. 访问抖音创作者页面
3. 显示登录二维码
4. **你需要**: 使用抖音 APP 扫码登录
5. 登录成功后自动保存 Cookie
6. 浏览器自动关闭

**预期结果：**
```
登录成功！Cookie 已保存
```

### 步骤 2：检查登录状态

```bash
python scripts/publish.py --check-login
```

**预期结果：**
```
✅ 已登录
```

### 步骤 3：发布视频

将你的测试视频复制到 `storage/videos/` 目录，然后运行：

```bash
python scripts/publish.py \
  --video ./storage/videos/test.mp4 \
  --title "我的第一个自动发布视频" \
  --desc "使用抖音发布工具自动上传 #AI 自动化 #测试"
```

**参数说明：**
- `--video`: 视频文件路径
- `--title`: 视频标题
- `--desc`: 视频描述（可以包含#话题标签）

**预期过程：**
```
1. 启动浏览器
2. 加载 Cookie（自动登录）
3. 导航到上传页面
4. 上传视频文件
5. 填写标题和描述
6. 点击发布按钮
7. 等待发布完成
```

**预期结果：**
```
✅ 发布成功！
```

## 🔍 故障排除

### 问题 1：登录超时

**现象**: 运行登录脚本后超时 180 秒

**原因**: 
- 网络问题
- 二维码过期
- 页面元素检测失败

**解决方法**:
```bash
# 1. 清除旧 Cookie
rm -rf storage/cookies/*

# 2. 重新登录
python scripts/quick_login.py

# 3. 确保网络通畅
# 4. 使用抖音 APP 快速扫码
```

### 问题 2：发布失败

**现象**: 发布时报错或卡住

**可能原因**:
1. Cookie 过期
2. 视频格式不支持
3. 视频太大
4. 网络问题

**解决方法**:
```bash
# 1. 重新登录
python scripts/quick_login.py

# 2. 检查视频
# - 格式：MP4
# - 大小：< 100MB
# - 时长：15-60 秒

# 3. 检查网络连接

# 4. 查看详细日志
python scripts/publish.py -v ./video.mp4 -t "标题" -d "描述" --verbose
```

### 问题 3：触发风控

**现象**: 发布时被阻止或需要验证

**原因**: 操作过于频繁，被抖音判定为异常行为

**解决方法**:
- 等待 24 小时后再试
- 降低发布频率
- 使用真实的网络环境
- 不要使用代理
